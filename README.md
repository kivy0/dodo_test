# Прототип системы детекции уборки столиков по видео

## 📦 Установка и запуск

### 1. Установить менеджер пакетов `uv`

```bash
# macOS / Linux
curl -LsSf https://astral.sh/uv/install.sh | sh

# Windows
powershell -ExecutionPolicy ByPass -c "irm https://astral.sh/uv/install.ps1 | iex"
```
### 2. Запустить

```bash
uv run main.py --video [Путь к видео]
```
uv автоматически создаст виртуальное окружение и установит все зависимости
из pyproject.toml при первом запуске.

### 3. Интерактивный выбор областей

При запуске откроются **3 окна** для ручного выделения областей (ROI):

| # | Окно | Что выделять |
|---|------|-------------|
| 1 | Select table area | Область стола на кадре |
| 2 | Select date area | Область с датой |
| 3 | Select time area | Область с временем |

Выделите область мышью и нажмите **Enter** для подтверждения.

## 🧠 Логика детекции событий

### Пространственная тепловая карта

Ключевая идея - **не просто считать людей в зоне**, а строить
пространственную тепловую карту поверх области стола:

1. **Область стола** разбивается на сетку ячеек заранее заданого размера.

2. **На каждом кадре:**
   - Тепло всех ячеек **затухает** с коэффициентом heat_decay
   - Для каждого обнаруженного человека проверяется **доля пересечения**
     его bounding box с областью стола overlap_threshold.
   - Если пересечение достаточное - в соответствующие ячейки сетки
     **добавляется тепло** heat_increment.
   - Значения ячеек ограничиваются диапазоном [0, 1].

3. **Определение статуса** по максимальному значению тепловой карты:

   - heatmap.max() >= порог                                      →  🔴 OCCUPIED (занят)
   - есть человек в зоне и доля пересечения > overlap_threshold  →  🟡 PASSING (проходит)
   - иначе                                                       →  🟢 FREE (свободен)

### Почему тепловая карта, а не простой счётчик?

| Проблема | Решение |
|----------|---------|
| Человек прошёл мимо стола | Тепло не успевает накопиться → статус PASSING |
| Кратковременная потеря детекции | Тепло затухает плавно → статус не мигает |
| Несколько людей за столом | Тепло накапливается в разных ячейках хотя могут быть и пересечения |
| Человек стоит на краю зоны | overlap_threshold фильтрует случайные пересечения |

### OCR-распознавание времени

- **Дата** считывается один раз с первого кадра (формат YYYY-MM-DD)
- **Время** считывается с каждого кадра (формат HH:MM:SS)
- Предобработка: бинаризация → эрозия/дилатация → инверсия → рамка

## ⚠️ Известные проблемы

### 1. Быстрые множественные детекции
Расчет PASSING статуса реализован если в roi стола есть bbox человека хотя бы наполовину. Из чего следует PASSING довольно нестабильный, поэтому мы получаем довольно маленькое 9 секунд среднее ожидание по статусу PASSING. Если потребуется к нему можно применить туже логику тепла для стабилизации.

![Проблемный кадр - быстрые множественные детекции](images/problem1.png)


### 2. Долгое присутствие работника в зоне стола.

![Проблемный кадр - ложный статус OCCUPIED.](images/problem2.png)

Когда работник убирает стол, тепло успевает накопиться. Тоже самое случится, если человек не пройдет мимо стола а будет стоять рядом с ним.
